# 2026-06-15 实战复盘：主材归一化漏洞 + 汤锅冲突

## 事故1：MAIN_INGREDIENT_MAP 烹饪方法变体覆盖不足

### 现象
用户说"昨天吃过鸡了，怎么今天还在推荐？"
检查 history.json：6月14日有"毛豆烧鸡"
检查预检脚本输出：鸡肉**不在** excluded_main_ingredients 中

### 根因
recipe_preflight.py 的 MAIN_INGREDIENT_MAP 只有：
- "焖鸡" → "鸡肉"
- "豉油鸡" → "鸡肉"
- "白切鸡" → "鸡肉"
- "盐焗鸡" → "鸡肉"
- "手撕鸡" → "鸡肉"

缺少"烧鸡"、"炒鸡"、"烤鸡"、"炖鸡"等烹饪方法变体。
"毛豆烧鸡"提取时匹配不到任何条目 → 鸡肉漏进候选池。

### 修复
```python
# recipe_preflight.py line 51
"豉油鸡": "鸡肉", "白切鸡": "鸡肉", "盐焗鸡": "鸡肉", "手撕鸡": "鸡肉", "烧鸡": "鸡肉", "炒鸡": "鸡肉", "烤鸡": "鸡肉", "炖鸡": "鸡肉",
```

### 持续维护规则
- 每次发现漏掉的归一化条目，立即补到 MAIN_INGREDIENT_MAP
- 优先覆盖"烹饪方法+主材"格式（烧X/炒X/烤X/炖X/蒸X/煎X）
- 同类禽肉需覆盖变体：鸭（烧鸭/烤鸭/卤鸭）、鹅（烧鹅）、鸽（乳鸽/鸽子）

## 事故2：汤锅复用冲突

### 现象
厨师审核 subagent 发现：鸽子汤用"电压锅/汤锅"保温，白灼鲜虾用汤锅烧水，两菜共用汤锅 → 器具冲突。

### 根因
器具分布表写成"电压锅/汤锅"合并一行，掩盖了汤锅被两道菜同时需要的事实。
- 汤品 = 汤锅持续占用（小火保温）
- 白灼 = 汤锅阶段性占用（烧水+焯虾）

### 修复
1. 鸽子汤电压锅炖好后，泄压转移到砂锅保温 → 释放汤锅
2. 器具分布表分开列：电压锅、砂锅、汤锅各自独立
3. 人工器具使用顺序改为：汤锅（白灼虾）→ 洗锅 → 炒锅（茭白）→ 煎锅（青花鱼）

### 通用规则
- 方案中同时有"汤锅类汤品"+"需要汤锅焯水/白灼的菜" → 必须写明转移至砂锅释放汤锅
- 器具分布表必须分开列，不可合并写
- 如家里只有1个汤锅，白灼类菜必须改炒锅焯水

## lark-cli v2 创建文档方式变更

### 现象
`lark-cli docs +create --title "标题" --markdown -` 报错：v1 interface has been shut down

### 修复
改用 v2 API：`lark-cli docs +create --api-version v2 --doc-format markdown --content "$CONTENT"`
文档标题从 content 中的 `# 标题` 自动提取，不需要 `--title` flag。
