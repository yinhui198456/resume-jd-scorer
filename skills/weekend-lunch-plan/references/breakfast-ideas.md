# 早餐候选库

适用范围：`meal_type=breakfast`。本文件用于生成早餐候选池，重点满足“叮咚买菜可买到、半成品友好、预约或快手出菜”的个人偏好。

优先数据源：项目内 `data/breakfast_products.json`。该文件维护叮咚买菜风格早餐商品池，当前来源为截图识别和手工录入，后续可通过定时自动完善流程更新 `last_verified`、`keywords` 和候选项。

## 用户偏好

- 优先叮咚买菜等渠道常见的冷冻、冷藏、预制或半成品早餐。
- 优先不复杂处理、不长时间看火、不产生大量油烟的方案。
- 优先使用电蒸锅预约、豆浆机预约、煎锅快煎等方式快速出菜。

## 候选品类

### 主食 / 半成品

- 蒸点类：肉粽、烧麦、窝窝头、汤包、小笼包、花卷、玉米卷
- 煎烙类：煎饺、生煎、手抓饼
- 可搭配类：馄饨、饭团、贝果、杂粮粥、玉米、红薯

### 饮品

- 牛奶
- 无糖豆浆
- 米酒
- 玉米汁
- 米糊

### 蛋白与果蔬补充

- 蛋白：鸡蛋、奶酪、酸奶、虾仁、鸡胸肉片
- 果蔬：小番茄、黄瓜条、蓝莓、香蕉、橙子、苹果

## 快手组合模板

### A. 电蒸锅预约

- 主食：烧麦 / 肉粽 / 花卷 / 小笼包 / 汤包 / 窝窝头
- 蛋白：水煮蛋 / 鸡蛋羹 / 奶酪
- 饮品：牛奶 / 无糖豆浆
- 果蔬：小番茄 / 黄瓜条 / 蓝莓

适合前一晚放入蒸锅预约，早上只需取出和简单摆盘。

### B. 豆浆机预约

- 主食：窝窝头 / 玉米卷 / 花卷 / 肉粽
- 蛋白：鸡蛋 / 酸奶 / 奶酪
- 饮品：无糖豆浆 / 玉米汁 / 米糊
- 果蔬：香蕉 / 苹果 / 蓝莓

适合豆浆机夜间预约，主食用蒸锅或微波炉同步处理。

### C. 煎锅快煎

- 主食：煎饺 / 生煎 / 手抓饼
- 蛋白：鸡蛋 / 奶酪 / 酸奶
- 饮品：牛奶 / 无糖豆浆
- 果蔬：黄瓜条 / 小番茄 / 橙子

适合 10-15 分钟完成。煎锅类只选 1 个主食，不要三套方案都使用高油煎炸。

## 健康边界

- 控制油糖：生煎、煎饺、手抓饼属于偏油选项，每次最多作为一套方案主食。
- 肉粽通常油盐较高，应搭配牛奶/无糖豆浆和清爽果蔬，不再叠加煎炸主食。
- 米酒注意糖分，默认少量搭配，不作为儿童主要饮品。
- 早餐仍需满足 `1 主食 + 1 蛋白 + 1 饮品 + 1 果蔬`，不能只给主食拼盘。

## 商品池维护规范

`data/breakfast_products.json` 每条记录必须保留：

- `name`：商品或稳定品类名，例如烧麦、小笼包、牛奶
- `category`：主食 / 蛋白 / 饮品 / 果蔬
- `source`：默认叮咚买菜
- `method`：电蒸锅预约 / 豆浆机预约 / 煎锅快煎 / 即食
- `meal_type`：固定 breakfast
- `health_tags`：快手、低糖、高钙、儿童友好等
- `risk_note`：油盐糖、儿童饮用、搭配边界
- `update_mode`：manual_screenshot / manual_entry / future_scheduled_enrichment
- `last_verified`：最近确认日期
- `keywords`：后续检索或定时自动完善使用

后续如果接入自动完善，不直接依赖叮咚账号实时登录；优先使用用户提供截图、导出清单或人工确认后的候选项更新本地商品池。
