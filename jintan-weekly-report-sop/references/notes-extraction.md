# 备注列日期前缀提取逻辑

## 场景

在线台账【备注】列通常用日期前缀记录历史更新，如：
```
0421:日常医疗相关指标口径已梳理并反馈医管中心，暂无反馈
0427:
1、公卫部分的调研据客户反馈得排到五一后
2、医管中心回复没有空看
0429:已给出一版反馈，财务部分已经和肖科确认
0506:101办公室，当面沟通
0509:医管、公卫已进行调研，并输出指标口径，待与信息科进一步核对确认
```

## 提取规则

`report_engine_v9.py` 的 `clean_notes()` 方法已实现：

1. **识别格式**：匹配 `MMDD:` / `MM/DD:` / `MM-DD:` 前缀（正则 `^(\d{2})[\/\.-]?(\d{2})[:：]\s*(.*)`）
2. **本周优先**：遍历所有行，找到落在报告周（周一至周日）内日期**最大**（最新）的一条
3. **内容剥离**：返回时去掉日期前缀，只保留内容（如 `医管、公卫已进行调研...`）
4. **回退机制**：若无本周更新，取最后一行并剥离旧日期前缀

## 代码位置

`scripts/report_engine_v9.py` → `ReportBuilder.clean_notes()`

## 关键点

- 报告周范围：`self.monday` 至 `self.sunday`（含周六日，覆盖补班场景）
- 多条本周更新时取日期最大的（如同时有 0506 和 0509，取 0509）
- 旧日期前缀（如 0421、0427）会被自动跳过，不会出现在周报中
