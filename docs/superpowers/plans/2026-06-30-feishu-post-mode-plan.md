# Feishu Post Message Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `CTI_FEISHU_MESSAGE_STYLE=post` mode to `claude-to-im` so normal Feishu replies use `msg_type: post` (readable + mobile-copyable), while skipping streaming cards.

**Architecture:** Modify the upstream Feishu adapter (`/root/.codex/skills/Claude-to-IM`) to recognize a new `post` message style. When active, the adapter sends all normal replies via `sendAsPost()` and disables CardKit streaming. Build upstream, build the skill wrapper, update both runtime configs, restart both bridges, and save a patch file.

**Tech Stack:** TypeScript, Node.js 22, esbuild, Feishu/Lark SDK, Git.

## Global Constraints

- Two runtime homes share the same installed skill code via symlinks.
- Secrets must stay in `config.env` and must never be logged or committed.
- Both Codex and Claude Code runtimes must behave identically after this change.
- `CTI_AUTO_APPROVE=true` is required because Feishu permission cards are unreliable in this deployment.
- After any source change, run `npm run build` in both the upstream repo and the skill wrapper repo.

  it('sends post message when bridge_feishu_message_style is post', async () => {
    setupContext({ bridge_feishu_message_style: 'post' });
    const calls: Array<{ method: string; payload: any }> = [];
    const client = {
      im: {
        message: {
          create: async (payload: any) => {
            calls.push({ method: 'message.create', payload });
            return { code: 0, data: { message_id: 'message-post' } };
          },
        },
      },
    };

    const adapter = new FeishuAdapter() as any;
    adapter.restClient = client;

    const result = await adapter.send({
      address: { channelType: 'feishu', chatId: 'chat-1', userId: 'user-1' },
      text: '**加粗标题**\n\n- 列表项 A\n- 列表项 B',
      parseMode: 'Markdown',
    });

    assert.equal(result.ok, true);
    assert.equal(calls[0].payload.data.msg_type, 'post');
    const postContent = JSON.parse(calls[0].payload.data.content);
    assert.equal(postContent.zh_cn.content[0][0].tag, 'md');
    assert.equal(postContent.zh_cn.content[0][0].text.includes('加粗标题'), true);
  });
```

  it('does not create streaming cards when bridge_feishu_message_style is post', async () => {

    const calls: Array<{ method: string; payload: any }> = [];
    const client = {
      cardkit: {
        v1: {
          card: {
            create: async (payload: any) => {
              calls.push({ method: 'card.create', payload });
              return { code: 0, data: { card_id: 'card-1' } };
            },
          },
          cardElement: {
            content: async (payload: any) => {
              calls.push({ method: 'cardElement.content', payload });
              return { code: 0, data: {} };
            },
          },
        },
      },
      im: {
        messageReaction: {
          create: async (payload: any) => {
            calls.push({ method: 'messageReaction.create', payload });
            return { code: 0, data: { reaction_id: 'reaction-1' } };
          },
        },
      },
    };

    const adapter = new FeishuAdapter() as any;
    adapter.restClient = client;
    adapter.lastIncomingMessageId.set('chat-1', 'incoming-1');

    adapter.onMessageStart('chat-1');
    adapter.onStreamText('chat-1', 'partial response');
    adapter.onToolEvent('chat-1', [{ id: 'tool-1', name: 'tool', status: 'running' }]);
    const finalized = await adapter.onStreamEnd('chat-1', 'completed', 'final response');
    await new Promise((resolve) => setImmediate(resolve));

    assert.equal(finalized, false);
    assert.equal(calls.some((call) => call.method === 'card.create'), false);
    assert.equal(calls.some((call) => call.method === 'cardElement.content'), false);
  });
```
