# 飞书 Thread 引用错位 → 误拒答题 (2026-06-18)

## 症状
用户回复 "B" 判 Q24，但飞书消息引用了 Q23 的旧消息。
Agent 看到引用错位，向用户确认并要求重新作答。
用户反馈："24题已经回答了，怎么又出了？"

## 根因
飞书 "Replying to" 引用经常自动指向旧消息（用户点回复时的 UI 行为），但用户实际是针对**最新展示的题目**作答。单字母答案 A/B/C/D 是明确的，不需要确认。

## 修复
"答题前置三查" 规则 #3 修订：
- 单字母回复 → 直接用 current_qid 判题，不问
- 回复内容包含题干/选项且与 current_qid 不符 → 才需要确认

## 教训
飞书引用错位 ≠ 用户意图不明。单字母答案 + state 的 current_qid 已足够确定用户要答哪题。
