# P0-P3 四重防护体系（2026-06-07 实施）

## 背景
LLM 在刷题流程中反复出现：串题（改写出题）、编题（COMPLETE 后自创）、判题错位（基于错误题目判定）。根因：LLM 是生成器非转发器，指令约束是软强制。

## 四重修复

### P0：零 LLM 参与出题/判题
- **目标**：消除 LLM 在出题→判题链路中的编辑机会
- **实现**：`quiz_bot.py` 判题输出含完整正确选项文本，LLM 只做转发
- **状态**：cron prompt 已更新（禁止补充解析、禁止改写选项文本）

### P1：精简 Skill 指令
- **目标**：核心指令不被噪音淹没
- **实现**：`quiz-bot/SKILL.md` 从 476 行精简至 72 行（-85%）
- **删除**：重复规则 3 遍→1 条、过期脚本引用（update_daily_tracking.py、quiz_session.py）、矛盾指令（auto_next 全篇清除）、死引用（deep_audit.py、quality_audit.py）

### P2：Hash 一致性校验（防串题核心）
- **目标**：检测 LLM 是否改写了题目内容
- **实现**：
  - `quiz_bot.py` 的 `_output_question()` 生成题目内容 MD5 hash（前8位）存入 `quiz_bot_state.json` 的 `q_hash` 字段
  - `cmd_answer()` 判题时比对当前 hash 与 state 中的 hash
  - 不匹配 → 输出 `⚠️ 题目内容已变更！请重新运行 next` 并拒绝判题
- **验证**：4/4 测试通过（正常判题✅、hash篡改拒绝✅、hash恢复✅、答错输出选项文本✅）

### P3：判题输出含完整正确选项文本
- **目标**：防止 LLM 自由解释答案含义
- **实现**：`cmd_answer()` 的 md 格式输出增加正确选项完整文本
  - 格式：`D 选项：Agent 间通过灵活 Handoff 机制根据对话内容在专业 Agent 间无缝切换控制权`
- **效果**：LLM 判题后不再有"解析空间"，直接转发即可

## 测试验证

```
Test 1: 正常出题→判题 → ✅ 正常通过
Test 2: 篡改 hash 后判题 → ✅ 拒绝并警告
Test 3: 恢复 hash 后判题 → ✅ 恢复正常
Test 4: 答错时也输出正确选项文本 → ✅ 确认输出
```

## 已知限制
- hash 校验是检测手段而非预防手段：LLM 若改写后碰巧 hash 仍匹配（极罕见），无法拦截
- 终极方案：P0 架构升级（脚本→飞书直达，全自动流水线），彻底消除 LLM 中转环节
- quiz_bot_state.json 全局共享，多并发会话会互相覆盖（需 session 级隔离）

## 相关文件
- `quiz_bot.py` — 核心引擎，含 P1 defensive、P2 hash 校验、P3 完整选项文本
- `quiz_bot_state.json` — 状态文件，含 `q_hash` 字段
- `quiz-bot/SKILL.md` — 72 行精简版
