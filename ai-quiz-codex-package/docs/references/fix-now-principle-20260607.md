# 质量优先与“立即修复”原则 (2026-06-07)

## 核心原则
**有问题现在就修，绝不拖延到“今晚”。**

## 背景
在一次刷题中，Agent 发现了出题逻辑漏洞（模块结束后会空白），为了“不打断聊天”，Agent 提议“今晚重构代码再修，现在先用软约束顶着”。
用户强烈反对：「不是太理解，为什么是今晚改，而不是现在？有问题为什么不改？」

## 教训
1. **拖延修复是风险累积的根因**：“先凑合用，晚上修”的思维模式导致了之前反复出现的串题、编题事故。
2. **质量 > 流程顺畅**：哪怕修代码需要 10 分钟、打断当前对话，也比带着风险继续跑强。
3. **行动指南**：
   - 发现数据不一致 → **立即**运行对齐脚本。
   - 发现代码逻辑漏洞 → **立即**打补丁。
   - 发现进度丢失 → **立即**补录 tracking 记录。
