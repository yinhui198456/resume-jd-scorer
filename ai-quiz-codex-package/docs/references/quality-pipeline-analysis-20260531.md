# 题库质量管道分析（2026-05-31 交叉验证）

## 核心发现

**GitHub 原始数据不含 4 选项** — agent-interview-hub 等仓库的题目是纯问答题（题干 + 段落答案），没有 A/B/C/D 结构。选项由 LLM 脚本补全，质量从源头不可靠。

## 破坏性脚本 → 具体质量问题

### 🔴 quick_fix_quality.py（最严重）
- `FILLERS` 列表 62 条填充语被追加到任何短干扰项
- "这是 Prompt Engineering 的思路，非架构层面的方案" 出现在 183 个选项、66 道题中
- 74 道题的 2+ 个选项末尾有同一句套话
- **影响**：干扰项变得极其明显（结构相同），一眼可排除

### 🔴 deep_rebuild_bank.py
- `generate_distractors()` 用 15 个硬编码模板生成干扰项
- 模板如「该技术的核心是增加模型的训练数据量」「主要通过减少模型层数来提高推理效率」
- **影响**：109 题（15% 题库）有明显伪造的、与题干无关的干扰项

### ⚠️ fix_length_bias.py
- 追加「，但这仅适用于特定场景」等空洞后缀
- **影响**：长度比改善，但语义空洞

## 质量审计结果

| 维度 | 不合格数 |
|---|---|
| answer/explanation 一致性 | 258 |
| 长度偏见 | 172 |
| filler 密度 | 104 |
| 干扰项语义多样性 | 74 |
| 选项重复 | 45 |

**192 题通过（27.3%），511 题不合格（72.7%）**

## 质量门控（已启用）

`quiz_bot.py` 集成的 `quality_gate.py` 在出题时自动跳过 511 道不合格题。
- 通过条件：综合评分 ≤ 25 且所有单维度 ≤ 50
- 降级模式：审计文件不存在时 fail-open
- 用户看到 "⚠️ 已跳过 N 道质量不合格的题目" 提示

## 经验教训

1. **每次「修复」都在制造新 bug** — 因为脚本不理解语义，只做文本追加/替换
2. **只有手工逐题重写的脚本真正改善了质量**（phase1_fix_bias.py, fix_distractors_v4.py）
3. **质量过滤器比修复更可靠** — 过滤 511 题比尝试修复 511 题更安全
4. **用户说「反复改了100版」时，停止修补，做根因分析**
